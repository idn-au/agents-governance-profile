Preamble ↑

Abstract ↑

This is the specification document of the Agents Governance Profile which is a Profile of several Semantic Web / Knowledge Graph models (ontologies) for the representation of sophisticated Agent (people and organisations) relations, mainly for the purposes of the governance of Entities (things), in particular data.

This document specifies requirements of data that must be met for that data to claim conformance to the Agents Governance Profile. Data that conforms to this profile necessarily conforms to those Profiles and Standards that this Profile extends and constrains.

This specification document is not to be used for the automated testing of conformance of RDF data to this profile. That role belongs to the machine-readable validation resources:

  • SHACL Validator
  • Compounded Validator

The first validator above contains tests for this profile's requirements only. The second contains this profile's tests as well as those from the other Profiles and Standards that this profile, profiles.

For the list of all resources within this profile, see the profile definition:

  • Agents Governance Profile Definition

1. Introduction ↑

The Agents Governance Profile is a Profile of several well-known ontologies that represent Agents - People and Organisations - and their relationships to Entities - things. This document, the profile's Specification defines the rules for modelling Agent → Agent & Agent → Entity relations.

1.2 Qualified Relations Pattern

The Agent → Entity & Agent → Agent relations modelled by this Specification are done so using the qualified relations pattern which is described for RDF modelling in the Linked Data Patterns online textbook but which are summarised like this:

Figure Qualified: Where Resource A has a relation to Resource B and that relation is described by a role, Resource C, a pattern that interjects an anonymous node between A ∓ B can be made that links the via a relationship to C. This figure shows Resources A linked to an anonymous node [ ] by property x which then links to the target Resource B and the qualifying Resource C.

All of the major relationships between Agents & Entities in this Specification use qualified relations patterns taken from the Provenance Ontology [PROV] or the Data Catalogue Vocabulary [DCAT].

1.3 Use of vocabularies

The definitions of the roles used in the qualified relations roles that Agents take with respect to Entities and other Agents must be formalised in some way and this Profile requires the use of vocabularies of roles for this, in particular the International Organization for Standardization (ISO)'s Role Codes vocabulary (for Agent → Entity) and this Profile's own Agent Delegations Role Codes vocabulary (for Agent → Agent). Users of this Profile are encouraged to extend these vocabularies, or create their own, to capture roles of interest to them.

1.4 Origins of this Profile

This Profile's relationships are general in nature - they can be used for many purposes - however the specific motivation for this work was to represent indigenous peoples' relationships, or lack of them, to data by or about them. For this reason, the Indigenous Data Network, the publisher of this Profile, uses an extended form of the ISO Role Codes vocabulary:

  • IDN Role Codes

Structure of this Specification

The Requirements section following this one defines the individual modelling patterns and element values that must be used by data wishing to conform to this Profile.

Following that, the Example section gives a data example implementing all the individual requirements.

Finally, the Mappings annex provides mapping from some of the elements and patterns described here to other modelling systems.
