Preamble ↑

Abstract ↑

This is the specification document of the Agents Governance Profile which is a Profile of several Semantic Web / Knowledge Graph models (ontologies) for the representation of sophisticated Agent (people and organisations) relations, mainly for the purposes of the governance of Entities (things), in particular data.

This document specifies requirements of data that must be met for that data to claim conformance to the Agents Governance Profile. Data that conforms to this profile necessarily conforms to those Profiles and Standards that this Profile extends and constrains.

This specification document is not to be used for the automated testing of conformance of RDF data to this profile. That role belongs to the machine-readable validation resources:

  • SHACL Validator
  • Compounded Validator

The first validator above contains tests for this profile's requirements only. The second contains this profile's tests as well as those from the other Profiles and Standards that this profile, profiles.

For the list of all resources within this profile, see the profile definition:

  • Agents Governance Profile Definition

An agent is something that bears some form of responsibility for an activity taking place, for the existence of an entity, or for another agent's activity.

1. Introduction ↑

The Agents Governance Profile is a Profile of several well-known ontologies that represent Agents - People and Organisations - and their relationships to Entities - things. This document, the profile's Specification defines the rules for modelling Agent → Agent & Entity → Agent relations.

1.2 Qualified Relations Pattern

The Entity → Agent & Agent → Agent relations modelled by this Specification are done so using the qualified relations pattern which is described for RDF modelling in the Linked Data Patterns online textbook but which are summarised like this:

Figure Qualified: Where Resource A has a relation to Resource B and that relation is described by a role, Resource C, a pattern that interjects an anonymous node (a Blank Node) between A & B can be made that links them via a relationship to C. This figure shows Resource A linked to an anonymous node, [ ], by property x which then links to the target Resource B and the qualifying Resource C.

All of the major relationships between Agents & Entities in this Specification use qualified relations patterns taken from the Provenance Ontology [PROV] or the Data Catalogue Vocabulary [DCAT].

1.3 Use of vocabularies

The definitions of the roles used in the qualified relations roles that Agents take with respect to Entities and other Agents must be formalised in some way and this Profile requires the use of vocabularies of roles for this, in particular the International Organization for Standardization (ISO)'s Role Codes vocabulary (for Entity → Agent roles). Users of this Profile are encouraged to extend these vocabularies, or create their own, to capture roles of interest to them.

1.4 Origins of this Profile

This Profile's relationships are general in nature - they can be used for many purposes - however the specific motivation for this work was to represent indigenous peoples' relationships to data.

Recognising the generic nature of this Profile, the Indigenous Data Network, the publisher of this Profile, is very happy to consider updates to this Profile that others, including non-Australian indigenous groups, propose.

Of primary concern for Australian indigenous peoples is data about them, so the IDN has extended the ISO Role Codes vocabulary to contain subject agent and subject agent representative roles in defined in their IDN Role Codes vocabulary. These roles are also likely of general interest.

1.5 Structure of this Specification

  • The Requirements section following this one defines the individual modelling patterns and element values that must be used by data wishing to conform to this Profile.
  • Following that, the Example section gives a data example implementing all the individual requirements.
  • Finally, the Mappings annex provides mapping from some of the elements and patterns described here to other modelling systems.

2. Requirements ↑

The requirements for data to conform to this profile are listed here. They are organised into the following subsections:

  • Agent → Agent Relations - indicating how Agents relate to other Agents
  • Entity → Agent Relations - indicating how Agents relate to Entitys

2.1 Agent → Agent Relations

Req AA1: Relationships between Agent instances MUST be modelled using the Qualified Relations Pattern with the first Agent indicating a dcat:Relationship object with the dcat:relation property which, in turn, indicates the second Agent with the prov:agent property and the role for the relationship, as a skos:Concept, with a dcat:hadRole property.

Figure AA1 below gives the general pattern and a specific example of data conforming to Req AA1.

Figure AA1: A. The general Qualified Relations Pattern to be followed for Agent/Agent relations. B. A figure of the data in Example AA1 below.

Example AA1: Person X is shown related to Organisation Y using the Qualified Relations Pattern <http://example.com/agent/person-x> a sdo:Person ; sdo:name "Person X" ; dcat:relation [ a dcat:Relationship ; prov:agent <http://example.com/agent/organisation-y> ; dcat:hadRole aarr:memberOf ; ] ; . <http://example.com/agent/organisation-y> a sdo:Organization ; sdo:name "Organisation Y" ; . <http://example.com/agent-relations/memberOf> a skos:Concept ; skos:prefLabel "Member Of"@en ; .

Req AA2: The role of Agent / Agent relations SHOULD be indicated with terms from this profile's Agent to Agent Relations Roles vocabulary or vocabularies extending it.

2.1.2 Entity → Agent Relations

Req EA1: Relationships between Entity instances and Agent instances MUST be modelled using the Qualified Relations Pattern with the Entity indicating a prov:Attribution object with the prov:qualifiedAttribution property which, in turn, indicates the Agent with the prov:agent property and the role for the relationship, as a skos:Concept, with a dcat:hadRole property.

Figure EA1 below gives the general pattern and a specific example of data conforming to Req EA1.

Figure EA1: A. The general Qualified Relations Pattern to be followed for Entity/Agent relations. B. An example of an E/A relation for the Indigenous Locations Dataset as per the example EA1 below.

Example EA1: Indigenous Locations Dataset, an Entity, is shown related to the Australian Bureau of Statistics, an Organisation, using the Qualified Relations Pattern <https://linked.data.gov.au/dataset/asgsed3/ILOC> a dcat:Dataset ; sdo:name "Indigenous Locations Dataset" ; prov:qualifiedAttribution [ a prov:Attribution ; prov:agent <https://linked.data.gov.au/org/abs> ; dcat:hadRole isorole:custodian ; ] ; . <https://linked.data.gov.au/org/abs> a sdo:Organization ; sdo:name "Australian Bureau of Statistics" ; . isorole:custodian; a skos:Concept ; skos:prefLabel "Custodian"@en ; .

Req EA2: The role of Entity / Agent relations SHOULD be indicated with terms from the International Organization for Standardization (ISO)'s Role Codes vocabulary or vocabularies extending it.

Annex A. Vocabularies↑

This Annex lists vocabularies relied on by this Profile: both those developed specifically for this Profile and others.

A.1. Agent to Agent Relations Roles Vocabulary↑

The Agent to Agent Relations Roles Vocabulary (AARR) was developed as part of this profile. It is online in authoritative form at:

  • https://w3id.org/idn/def/aarr

This vocabulary contains roles for relationships between Organisations, Persons and Persons & Organisations and is used in the Agent / Agent qualified relations pattern.

Some of the roles are:

  • Member: Person → Organisation
  • Part Of: Organisation → Organisation
  • Descendant Organisation: Organisation → Organisation
  • Proxy: Person → Person

A.2. Entity to Agent Relations Roles Vocabulary↑

The Entity to Agent Relations Roles Vocabulary suggested for use with this profile is the:

  • IDN Role Codes

This vocabulary contains roles for relationships between Entities (things) and Agents (People & Organisations) and is used in the Entity / Agent qualified relations pattern.

Some of the roles are:

  • Custodian: Person → Organisation
  • Rights Holder: Organisation → Organisation
  • subjectAgentRepresentatitve: Organisation → Organisation

This Annex defines mappings between this Profile and other Agent modelling systems.
